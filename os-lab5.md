## Завдання для попередньої підготовки
### Glossary of Basic English Terms 

**Filesystem – a hierarchical structure used by an operating system to organize and store files and directories.**

**Root directory (/) – the top-level directory in Linux from which all other directories originate.**

**Home directory (~) – a personal directory assigned to a user account where personal files and settings are stored.**

**Current working directory – the directory in which the user is currently located in the command-line interface.**

**pwd (print working directory) – a command that displays the full path of the current working directory.**

**cd (change directory) – a command used to navigate between directories in the filesystem.**

**ls (list) – a command used to display the contents of a directory.**

**ls -l – an option of the ls command that displays directory contents in long format, including permissions, owner, size, and modification date.**

**ls -a – an option of the ls command that displays all files, including hidden files.**

**mkdir (make directory) – a command used to create a new directory.**

**touch – a command used to create an empty file or update the timestamp of an existing file.**

**cp (copy) – a command used to copy files or directories from one location to another.**

**cp -r (recursive copy) – a command used to copy directories and their contents recursively.**

**mv (move) – a command used to move files or directories and also to rename them.**

**rm (remove) – a command used to delete files.**

**rm -r (recursive remove) – a command used to delete directories and their contents recursively.**

**rmdir – a command used to delete an empty directory.**

**Globbing – a shell mechanism used to match filename patterns using wildcard characters.**

**Wildcard - special characters such as `*`, `?`, and `[]` used to match patterns in filenames.**

**Pipe (|) – a shell operator that passes the output of one command as input to another command.**

**Redirection (>, >>) – a shell feature used to redirect command output into a file, where “>” overwrites and “>>” appends to the file.**

## Відповіді на теоретичні питання

### Порівняння файлових структур Windows-подібної та Linux-подібної системи
Файлова структура операційних систем Windows і Linux побудована за ієрархічним принципом, але між ними є певні відмінності.

У Windows використовується система дисків, які позначаються літерами (C:, D:, E: тощо). Кожен диск має власну окрему структуру папок. Для користувача верхнім рівнем зазвичай є «This PC» або «My Computer», де відображаються всі доступні диски.

У Linux застосовується інший підхід - тут існує одна єдина файлова система, яка починається з кореневого каталогу, що позначається символом «/». Усі файли та каталоги знаходяться всередині цієї структури. У Linux немає букв дисків, оскільки фізичні пристрої підключаються (монтуються) до певних каталогів у межах загальної системи.

Також у Linux структура каталогів є стандартизованою (наприклад, /home, /etc, /usr), що робить систему більш впорядкованою. У Windows розташування системних файлів може відрізнятися залежно від версії операційної системи.

Отже, основна різниця полягає в тому, що Windows працює з окремими дисками, позначеними літерами, а Linux має єдину файлову структуру з коренем «/».

### Поняття FHS та його використання
FHS (Filesystem Hierarchy Standard) - це стандарт, який визначає структуру каталогів у Linux-подібних операційних системах.

Метою FHS є уніфікація розташування системних файлів і каталогів, щоб структура файлової системи була однаковою або максимально схожою у різних дистрибутивах Linux.

Наприклад:

/home - домашні каталоги користувачів

/etc - конфігураційні файли системи

/bin - основні виконувані файли

/usr - програми та бібліотеки користувача

/var - змінні дані (логи, кеш тощо)

Завдяки стандарту FHS адміністратори та користувачі можуть легко орієнтуватися в будь-якій Linux-системі, незалежно від дистрибутива.

### Основні команди для роботи з файлами та каталогами в Linux
Основними командами для роботи з файлами та каталогами в Linux є такі:

Для створення використовується команда touch, яка дозволяє створити порожній файл, та команда mkdir, за допомогою якої створюється новий каталог.

Для копіювання застосовується команда cp, що копіює файли, а також cp -r, яка дозволяє копіювати каталоги разом з усім їх вмістом.

Для переміщення або перейменування файлів і каталогів використовується команда mv. Вона може як переносити об’єкти в інше місце, так і змінювати їх назву.

Для видалення файлів застосовується команда rm, а для видалення каталогів разом із вмістом - rm -r. Якщо каталог порожній, його можна видалити за допомогою команди rmdir.

Усі ці команди є базовими та необхідними для роботи з файловою системою в операційній системі Linux.

### 1.2 Опрацювання команд Lab 7 – Navigating the Filesystem

| Назва команди | Її призначення та функціональність |
|---------------|------------------------------------|
| pwd | Визначає місце знаходження користувача у файловій системі, показує поточну робочу директорію (print working directory). |
| echo $HOME | Виводить шлях до домашнього каталогу користувача через змінну середовища HOME. |
| cd / | Команда cd здійснює перехід до кореневого каталогу файлової системи. |
| cd | Команда cd без аргументів повертає користувача до домашнього каталогу. |
| cd /home | Здійснює перехід до каталогу /home за допомогою абсолютного шляху. |
| cd ~ | Здійснює перехід до домашнього каталогу користувача за допомогою символу ~. |
| cd ~root | Намагається перейти до домашнього каталогу користувача root (може виникнути помилка доступу). |
| cd /usr/bin | Здійснює перехід до каталогу /usr/bin за допомогою абсолютного шляху. |
| cd /usr | Здійснює перехід до каталогу /usr за допомогою абсолютного шляху. |
| cd /usr/share/doc | Здійснює перехід до каталогу /usr/share/doc. |
| cd bash | Здійснює перехід до підкаталогу bash за допомогою відносного шляху. |
| cd .. | Здійснює перехід на один рівень вище у файловій структурі. |
| cd ../dict | Переходить на один рівень вище та далі у каталог dict. |
| ls | Виводить список файлів та каталогів у поточній директорії. |
| ls -a | Виводить усі файли, включаючи приховані (які починаються з крапки). |
| ls -l | Виводить детальну інформацію про файл або каталог (тип, права доступу, власник, розмір, дата зміни). |
| ls -R | Виводить вміст каталогу разом із вмістом усіх його підкаталогів (рекурсивно). |
| ls -d | Відображає лише сам каталог без виведення його вмісту (часто використовується з шаблонами). |
| ls -d /etc/s* | Виводить файли та каталоги, назви яких починаються з літери «s». |
| ls -d /etc/???? | Виводить файли, назви яких складаються рівно з чотирьох символів. |
| ls -d /etc/[abcd]* | Виводить файли, назви яких починаються з літер a, b, c або d. |
| echo ~ ~sysadmin ~root ~mail ~nobody | Виводить шляхи до домашніх каталогів користувачів шляхом розгортання символу ~. |

### 1.3 Опрацювання команд Lab 8 – Managing Files and Directories

| Назва команди | Її призначення та функціональність |
|---------------|------------------------------------|
| echo * | Виводить усі файли та каталоги у поточній директорії (символ * відповідає нулю або більше символів). |
| echo D* | Виводить файли, назви яких починаються з літери D. |
| echo P* | Виводить файли, назви яких починаються з літери P. |
| echo *s | Виводить файли, назви яких закінчуються на літеру s. |
| echo D*n*s | Виводить файли, назви яких починаються з D та містять у середині символи n і s. |
| echo ?????? | Виводить файли, назви яких складаються рівно з шести символів (кожен ? відповідає одному символу). |
| echo D???????? | Виводить файли, назви яких починаються з D та складаються рівно з дев’яти символів. |
| echo ?????*s | Виводить файли, назви яких містять щонайменше шість символів і закінчуються на s. |
| echo [DP]* | Виводить файли, назви яких починаються з D або P (набір символів у квадратних дужках). |
| echo [!DP]* | Виводить файли, назви яких не починаються з D або P (заперечення набору через !). |
| echo [D-P]* | Виводить файли, назви яких починаються з літер у діапазоні від D до P. |
| echo [!D-P]* | Виводить файли, назви яких не починаються з літер у діапазоні від D до P. |
| cp /etc/hosts hosts | Копіює файл /etc/hosts у поточну директорію з новою назвою hosts. |
| cp -v /etc/hosts hosts | Копіює файл з відображенням інформації про процес копіювання (verbose-режим). |
| cp -v /etc/hosts . | Копіює файл у поточну директорію (символ «.» означає поточний каталог). |
| cp -p hosts /home/sysadmin | Копіює файл зі збереженням атрибутів (прав доступу та дати зміни). |
| cp -p /etc/hosts ~ | Копіює файл у домашній каталог користувача зі збереженням атрибутів. |
| cp hosts newname | Створює копію файлу з новою назвою newname. |
| cp -R /etc/udev Myetc | Копіює каталог разом із його вмістом (рекурсивне копіювання). |
| mkdir Myetc | Створює новий каталог Myetc. |
| rm hosts | Видаляє файл hosts. |
| rm hosts newname | Видаляє декілька файлів одночасно. |
| rm -r Myetc | Видаляє каталог разом із його вмістом рекурсивно. |
| rmdir directory | Видаляє порожній каталог. |
| touch premove | Створює порожній файл premove. |
| mv premove postmove | Переміщує або перейменовує файл premove у postmove. |
| rm postmove | Видаляє файл postmove. |
